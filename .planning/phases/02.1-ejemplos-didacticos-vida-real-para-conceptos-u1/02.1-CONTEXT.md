# Phase 02.1: Ejemplos didacticos vida real para conceptos U1 - Context

**Gathered:** 2026-03-02
**Status:** Ready for planning

<domain>
## Phase Boundary

Agregar ejemplos de la vida real inline dentro de los componentes Topic11-14 existentes en `src/content/units/u1/concepts.tsx`. Los ejemplos hacen tangibles los conceptos abstractos del temario de U1 usando escenarios universitarios y de empresas tech reales. No se crean componentes nuevos de seccion ni se modifica la estructura de ConceptsSection.

</domain>

<decisions>
## Implementation Decisions

### Ubicacion de ejemplos
- Inline dentro de cada Topic (Topic11, Topic12, Topic13, Topic14), justo despues del concepto que ilustran
- Maxima granularidad: cada concepto clave tiene su ejemplo inmediatamente despues
- Usar `Callout type="example"` existente — sin crear variantes nuevas
- Ejemplos siempre completos, sin truncar ni collapsible

### Tipo de escenarios
- Mezcla de contexto universitario UTFSM y empresas tech internacionales
- Nombrar empresas reales directamente ("Netflix almacena su catalogo en...")
- Proporcion universitario/tech: Claude decide segun que ejemplo ilustra mejor cada concepto

### Interactividad
- Snippets SQL estaticos con syntax highlighting (no ejecutables)
- Mini-tablas HTML de datos ilustrativos (3-5 filas) donde aplique
- Diagramas simples a discrecion de Claude donde aporten significativamente

### Densidad
- Obligatorio: ejemplos para "Dato vs Informacion" (1.1) y "Archivos vs BD" (1.2)
- Resto: Claude decide la densidad optima por tema
- Largo del ejemplo: Claude decide segun complejidad del concepto
- Tono: coloquial-cercano, segunda persona ("Imagina que estas en Spotify buscando una cancion...")

### Claude's Discretion
- Proporcion exacta de ejemplos universitarios vs tech por tema
- Cuantos ejemplos por tema (minimo los 2 obligatorios mencionados)
- Largo de cada ejemplo segun complejidad del concepto
- Donde incluir diagramas simples
- Donde incluir mini-tablas de datos ilustrativos
- Implementacion del syntax highlighting para snippets SQL

</decisions>

<specifics>
## Specific Ideas

- Tono diferenciado: el contenido teorico es formal academico, los ejemplos son coloquiales-cercanos para generar contraste y engagement
- Los snippets SQL son de referencia visual, no ejecutables — el estudiante ya tiene el editor SQL y ejercicios para practicar
- Mini-tablas ilustrativas muestran datos concretos (ej: 3 filas de tabla ALUMNO para explicar PK/FK)

</specifics>
