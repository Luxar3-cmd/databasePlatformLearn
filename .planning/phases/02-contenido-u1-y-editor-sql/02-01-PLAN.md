Las 5 tablas con columnas (CUIDADO: evitar reserved keywords de AlaSQL — usar nombres en espanol):
- departamentos: id INT, nombre STRING, codigo STRING
- profesores: id INT, nombre STRING, apellido STRING, departamento_id INT
- alumnos: id INT, nombre STRING, apellido STRING, rol STRING
- asignaturas: id INT, codigo STRING, nombre STRING, creditos INT, departamento_id INT
- inscripciones: id INT, alumno_id INT, asignatura_id INT, semestre STRING, nota FLOAT

2. Crear `src/engine/sql.ts` — singleton AlaSQL:
- Variable `let initialized = false` para evitar doble inicializacion (Pitfall 3 de RESEARCH.md)
- `initDb()`: DROP TABLE IF EXISTS para cada tabla, CREATE TABLE, INSERTs individuales (AlaSQL no soporta multi-INSERT en una sola llamada)
- `resetDb()`: setea `initialized = false`, llama `initDb()`
- `executeQuery(sql)`: try/catch, detecta `typeof rows === 'number'` para DML (Pitfall 2), retorna QueryResult
- `translateError(err)`: mapa de patrones de error AlaSQL a mensajes en espanol:
  - 'not found' -> 'Tabla no encontrada. Verifica el nombre.'
  - 'Unexpected token' -> 'Error de sintaxis SQL. Revisa la consulta.'
  - 'Column' -> 'Columna no encontrada: [detalle]'
  - 'parse' -> 'No se pudo interpretar la consulta SQL.'
  - default -> 'Error SQL: [mensaje original]'

3. Datos seed estilo UTFSM (~50-60 registros totales):
- departamentos (5): Informatica (DI), Matematica (DM), Fisica (DF), Industrias (DII), Electronica (DE)
- profesores (8): nombres chilenos reales, distribuidos entre departamentos
- alumnos (12-15): nombres chilenos, roles tipo 202173XXX-X
- asignaturas (8-10): codigos reales (INF-239, MAT-021, FIS-110, IND-163, etc.), creditos variados, distribuidos entre departamentos
- inscripciones (20-25): distribuidas entre alumnos y asignaturas, semestres 2024-1 y 2024-2, notas entre 1.0 y 7.0

Los datos deben soportar queries interesantes: JOINs alumno-inscripcion-asignatura, GROUP BY con COUNT/AVG, filtros por departamento, ORDER BY nota, WHERE nota >= 4.0.

4. Validar ejecutando en un test manual (o script temporal):

- SELECT asig.codigo, COUNT(*) as total FROM inscripciones i JOIN asignaturas asig ON i.asignatura_id = asig.id GROUP BY asig.codigo
- INSERT + verificar rowCount retorno numerico
- resetDb() + verificar que SELECT retorna datos originales
