   **Topic11 — "1.1 Definicion de Bases de Datos"** (paginas 1-10 del PDF):
   - Callout definition: que es un Dato (hechos relacionados con personas, objetos, eventos)
   - Callout definition: que es Informacion (datos organizados para toma de decisiones)
   - Callout definition: que es una Base de Datos (conjunto de archivos de datos relacionados)
   - Definicion tecnica de BD (repositorio de archivos, integrar, compartir)
   - Definicion organizacional (datos operacionales, 3 niveles: operacional, tactico, estrategico)
   - Componentes del enfoque de BD: BD fisica, Diccionario de Datos, DBMS (con sus 3 funciones: DDL, DML, DCL), Programas de Aplicacion, Interfaz de Usuario, Herramientas CASE, Usuarios (finales, desarrolladores, DBA)
   - Implementacion: 3 etapas (modelamiento, diseno/creacion, uso)

   **Topic12 — "1.2 Enfoque de Archivos vs Enfoque de BD"** (paginas 4-8):
   - Enfoque de Archivos (por Agregacion): cada aplicacion tiene sus archivos, sin planificacion corporativa
   - Desventajas del enfoque de archivos (tabla o lista): redundancia no controlada, dependencia de datos, pobre estandarizacion, inconsistencia, problemas con el cliente, escasa posibilidad de compartir, baja productividad
   - Enfoque de BD: datos como recurso compartido, independientes del programa, control centralizado via DBMS
   - Tabla comparativa (decision locked: "tablas HTML estilizadas con filas alternadas"):
     | Criterio | Enfoque Archivos | Enfoque BD |
     - Redundancia, Dependencia, Estandarizacion, Compartir datos, Productividad, Consistencia
   - Ventajas del enfoque de BD (lista): vision centralizada, minimizacion redundancia, independencia datos, estandarizacion, integracion/seguridad, facilidades desarrollo

   **Topic13 — "1.3 Tipos de Bases de Datos"** (paginas 16-24):
   - Clasificacion por 6 criterios:
     1. Estructura de datos: Jerarquica, Reticular, Relacional, Orientada a Objetos, Multidimensional
     2. Nivel organizacional: Operacional (OLTP), Gestion (DW/Data Mart), Estrategica (OLAP/Data Mining)
     3. Tipo de dato: Estructurados (relacional), Agregados (multidimensional), Semiestructurados (documentos), No estructurados (NoSQL)
     4. Ubicacion copia principal: Memoria principal (1 nivel), Disco (2 niveles), Almacenamiento terciario (3 niveles)
     5. Numero de procesadores: Serial, Paralela (MC, NC, DC, AC)
     6. Numero de sitios: Centralizada, Distribuida (homogenea, heterogenea)
   - Seccion especial sobre BD Relacional: concepto de relacion (tabla), tupla (fila), atributo (columna), dominio, PK, FK, NULL
   - Tabla comparativa OLTP vs OLAP
   - Ejemplo SQL: CREATE TABLE con PK y FK

   **Topic14 — "1.4 Proceso de Diseno de BD"** (paginas 10-11, 24+):
   - BD en el desarrollo de SI: etapas Analisis, Diseno, Construccion, Implementacion, Mantenimiento
   - 3 etapas del trabajo con BD: Modelamiento (modelo conceptual), Diseno (modelo logico via DDL), Uso (DML por usuarios y aplicaciones)
   - Niveles organizacionales y tipos de SI:
     - Tabla de decisiones por nivel (Estrategico/Tactico/Operacional) con tipo de decision, modelo, fuente, exactitud, amplitud, frecuencia, rango, uso
   - Tipos de SI: OLTP (TPS, MIS) vs OLAP (DSS)
   - Data Warehouse: concepto, ETL, relacion con OLAP y Data Mining
   - Business Intelligence como concepto integrador

   IMPORTANTE: El contenido debe ser fiel al PDF del curso. Usar el mismo vocabulario y ejemplos. No inventar conceptos que no estan en el material. Adaptar redaccion para formato web (mas conciso que el PDF, pero sin perder informacion clave).
