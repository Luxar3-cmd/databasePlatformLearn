**QuizSection.tsx** — Componente wizard con estado local:
- Tipos: `type QuizPhase = 'answering' | 'feedback' | 'complete'`
- Estado: `questions` (shuffled), `currentIndex`, `selectedOption`, `phase`, `results: boolean[]`
- Fisher-Yates shuffle function inline (3 lineas). CRITICO: shufflear opciones junto con tracking de correcta usando `{ text, isCorrect }` pattern (ver RESEARCH.md Pattern 3)
- `useEffect` en mount: shuffle questions y opciones
- `resetQuiz()` function que resetea TODOS los estados atomicamente (currentIndex=0, selectedOption=null, phase='answering', results=[], mas nuevo shuffle)

**Flujo visual (locked decisions, NO desviarse):**
1. Pantalla pregunta: barra progreso arriba (`(currentIndex+1)/total` como width %), texto pregunta, 4 botones de opcion (border-zinc-700 unselected, border-blue-500 selected)
2. Al seleccionar opcion: se marca visualmente (blue). Boton "Confirmar" aparece
3. Al confirmar: phase='feedback', opciones cambian color (verde correcta, rojo si eligio incorrecta), texto "Correcto!" o "Incorrecto" + respuesta correcta + explicacion breve. Boton "Siguiente" aparece
4. Sin navegacion hacia atras (flujo lineal)
5. Pantalla final (phase='complete'): score "X de Y correctas", icono segun performance, resumen de preguntas incorrectas con la respuesta correcta y explicacion. Boton "Intentar de nuevo" llama resetQuiz()

**Estilos:** Dark mode (zinc-900/zinc-800), feedback verde/rojo del patron ExerciseCard, blue-500 accents, lucide-react icons (CheckCircle2, XCircle, RotateCcw, ArrowRight). Sin CSS modules, solo Tailwind.
